Project management, built the Frappe way

Vernon Project manages projects, work breakdowns, teams, and task progress through a structured, role-based workflow on the Frappe Framework — with priced Project Details, a four-stage Project Todo workflow, recurring todos, per-phase time tracking, and per-team data filtering.

What is Vernon Project?

Vernon Project is a project-management application built on the Frappe Framework (Frappe ≥ 15.0, Python ≥ 3.10). Projects are broken down into Project Details — each with pricing, estimates, and a glossary-based grouping — and tracked via Project Todo items that move through a four-stage workflow: Planned → Done → Checked By PL → Completed.

The app layers on role-based permissions (System Manager, Project Owner, Project Leader, Project Team, and from v0.4.0 Project Admin), per-team data filtering, whitelisted APIs for status and notes operations, recurring todos, per-phase time estimation and actual-time tracking, and several monitoring reports.

Key features

Projects

Each project carries a brand, timeline, owner, leader, and team, with auto-naming and Calendar/Gantt views.

Point-based scoring

Score todos via Group weighting and a Group Level scale, with late/early adjustments recorded per role in the Point Ledger.

Project Details with pricing

Break work into items with current condition, expected outcome, glossary grouping, and price − discount = total.

4-stage todo workflow

Todos advance one stage at a time through Planned, Done, Checked By PL, and Completed with permission checks.

Team management

Manage teams with Owner, Leader, and Member roles; owner and leader are auto-added to the team.

Glossary system

Define per-project terminology used to group Project Details, plus a multi-select glossary table on each detail.

Per-phase time tracking

Capture estimated and actual hours for every workflow transition, with auto-summed totals (v0.3.0).

Recurring todos

Set todos Daily, Weekly, or Monthly with a recurring-until limit; a daily job auto-creates occurrences (v0.1.0).

Role-based permissions

Per-team data filtering across Project, Project Detail, and Project Todo, including the Project Admin role (v0.4.0).
