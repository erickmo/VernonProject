Installation & Setup

Install Vernon Project onto a Frappe bench, then walk a project from first-time setup through todos, the four-stage workflow, and progress monitoring.

Prerequisites

Before installing, make sure you have:

  • A running Frappe bench on Frappe >= 15.
  • Python >= 3.10 (the app declares requires-python >= 3.10).
  • A Frappe site to install the app onto.

Installation

From your bench directory, fetch the app, register it with the bench, then install it on a site and run migrations.

bash
cd $PATH_TO_YOUR_BENCH
bench get-app $URL_OF_THIS_REPO --branch develop
bench install-app vernon_project
bench --site [site-name] install-app vernon_project
bench --site [site-name] migrate
Tip

Substitute the real values: replace $URL_OF_THIS_REPO with your repository URL and [site-name] with your site. The develop branch shown above is only an example — use whichever branch you intend to deploy.

First-time setup

Vernon Project organizes work around a few foundational records. Create these before your first project so the dropdowns are populated and details can be grouped correctly.

  1. Create Brands. Add one or more Brand records (your clients/brands) from the Brand list. Every project is tied to a brand.
  2. Create Glossary entries. Define your terminology with Glossary records. Each glossary is tied to a single project and is later used to group Project Details.

Create a project

With the foundations in place, create the project itself.

  1. Open the Project list and click New.
  2. Enter the Project Name and Brand.
  3. Set the Start Date and Deadline.
  4. Assign the Project Owner and Project Leader.
  5. Add any additional Team Members and a Goal, then Save.
Team membership is automatic

The owner and leader (and the Project Admin, if set) are auto-added to Team Members, and duplicate rows are removed on save — you do not need to add them by hand.

Start Date must precede the Deadline

If the start date is on or after the deadline, the save is rejected by validation. Set Start Date earlier than Deadline.

Add project details

Break the project into work items using Project Detail records.

  1. Open the saved Project and create a new Project Detail. When you start from the project link, the project field is auto-filled.
  2. Enter a Title and select a Grouping — a Glossary belonging to that project.
  3. Fill in Current Condition, Expected Outcome, and optionally Price / Discount and the SOW note (keterangan_di_sow).
  4. Save.
Grouping and pricing constraints

The selected Grouping and any glossaries must belong to the project on the detail. Pricing is also validated: Price must be >= Discount (the total is computed as price minus discount).

Add and assign todos

Each Project Detail tracks granular work as Project Todo rows.

  1. Open the Project Detail and go to the TODO tab.
  2. Add a row with To Do (description), Assigned To, Deadline, Estimated minutes, and optional Notes.
  3. Optionally set per-phase estimates (estimated_planned_to_done, estimated_done_to_checked, estimated_checked_to_completed); the total is auto-calculated.
  4. Save. Status defaults to ⚪️ Planned.
Assign to team members

The Assigned To user should be a member of the project team, since the workflow lets the assignee advance their own todos. See the workflow guide for the full lifecycle.

Project Admins cannot advance status

The Project Admin role can read, write, and create Project Detail and Project Todo records, but it is hard-blocked from advancing todo status. The block is enforced at both the API layer (update_status) and the DocType layer (validate_project_admin_status_update).

Fields lock once Done

Once a todo reaches 🟠 Done or ✅ Completed, its assigned_to, estimated, and deadline fields become read-only to protect the audit trail. See the workflow and API reference for details.

Monitor progress

Track delivery using the built-in reports and the auto-computed Project Detail rollups.

  1. Use the Todo Report for ongoing todos and the Progress Report for completed work history.
  2. Use the Project Todo Deadline Report, plus the Daily Assignment and Daily Performance reports, to track deadlines and team output.
  3. On each Project Detail, review the computed stats — Total Todo, Todo without Estimation, Total Estimated, and Total Remaining Estimated. The detail auto-flips to Completed when all its todos are done.
See the full report catalog
