Entity-Relationship Diagram

A bird's-eye view of the Vernon Project data model — all 14 DocTypes and the Link / child-table relationships between them. The diagram is generated from the DocType JSON definitions and is the visual companion to the DocType Reference.

The diagram

Crow's-foot notation: || = exactly one, o{ = zero-or-many, o| = zero-or-one. Child-table DocTypes (istable: 1) are reached only through their parent's Table field and never exist as standalone documents.

Project Todo is now a standalone DocType

Earlier versions modelled tasks as a todo child table inside Project Detail. Project Todo is now a standalone DocType that links its parent via a project_detail Link, and carries its own allocations child table (Project Todo Allocation) for per-day planning.

How to read it

  • The work spine. Project → Project Detail → Project Todo is the three-level work-breakdown chain. Each Project Todo splits its time across days via Project Todo Allocation rows.
  • Classification. A Project links a Brand (formerly Customer).
  • Scoring. A Project Todo links a Group and picks a level from that group's Group Level child rows; awarded points are recorded as Point Ledger entries, one per role (assignee / leader).
  • Glossary. A Project Detail may link a grouping Glossary and a glossaries multiselect; each Glossary is scoped to a Project. The grouping seeds a default scoring Group when present.
  • Standalone stubs. Scope of Work and Project Proposal are submittable skeletons outside the work chain (Scope of Work optionally links a Proposal).
